models 命令

发现、扫描和配置 AI 模型的完整工具集

命令概述

models 命令是 OpenClaw 的模型管理中枢，用于发现可用的 AI 模型、扫描本地或远程模型提供者、以及配置模型参数。它支持多种模型提供者，包括 OpenRouter、Anthropic、OpenAI、Google 等。

核心功能

  • 模型发现: 自动检测可用的模型提供者和模型
  • 性能基准: 测试不同模型的响应时间和质量
  • 配置管理: 设置默认模型、API 密钥和模型参数
  • 成本分析: 估算不同模型的使用成本
  • 本地模型支持: 集成本地运行的开源模型

子命令列表

  • models list - 列出所有可用的模型
  • models providers - 列出所有配置的模型提供者
  • models scan - 扫描并发现新的模型
  • models test <model> - 测试特定模型的性能
  • models config <provider> - 配置模型提供者
  • models default <model> - 设置默认模型
  • models cost <model> - 查看模型成本信息
  • models local - 管理本地模型

模型提供者配置

OpenRouter

# 配置 OpenRouter
openclaw models config openrouter --api-key sk-or-v1-xxxxxxxx

# 设置 OpenRouter 为默认提供者
openclaw config set agent.model "openrouter/auto"

Anthropic (Claude)

# 配置 Anthropic
openclaw models config anthropic --api-key sk-ant-xxxxxxxx

# 使用 Claude 3 Opus
openclaw config set agent.model "anthropic/claude-3-opus-20240229"

OpenAI (GPT)

# 配置 OpenAI
openclaw models config openai --api-key sk-xxxxxxxx

# 使用 GPT-4
openclaw config set agent.model "openai/gpt-4"

本地模型 (Ollama/LM Studio)

# 配置 Ollama
openclaw models config ollama --url http://localhost:11434

# 使用本地 Llama 2 模型
openclaw config set agent.model "ollama/llama2"

常用操作示例

查看可用模型

# 列出所有模型
openclaw models list

# 列出特定提供者的模型
openclaw models list --provider openrouter

# 显示模型详细信息
openclaw models list --verbose

测试模型性能

# 测试单个模型
openclaw models test gpt-4

# 比较多个模型
openclaw models test gpt-3.5-turbo gpt-4 claude-2

# 测试带自定义提示
openclaw models test gpt-4 --prompt "Explain quantum computing in simple terms"

设置默认模型

# 全局默认模型
openclaw models default gpt-4

# 为特定代理设置默认模型
openclaw agents create dev-team --model gpt-3.5-turbo

# 临时覆盖默认模型
openclaw agent --model claude-2 --message "Summarize this article"

模型别名和简写

OpenClaw 支持模型别名，简化常用模型的引用：

别名 完整模型 ID 说明
gpt-4 openai/gpt-4 OpenAI GPT-4
gpt-3.5 openai/gpt-3.5-turbo OpenAI GPT-3.5 Turbo
claude-2 anthropic/claude-2 Anthropic Claude 2
claude-3-haiku anthropic/claude-3-haiku-20240307 Anthropic Claude 3 Haiku
auto openrouter/auto OpenRouter 自动选择最佳模型

高级配置

模型参数调优

# 设置温度参数
openclaw config set agent.temperature 0.7

# 设置最大 token 数
openclaw config set agent.max_tokens 2000

# 设置 top_p 参数
openclaw config set agent.top_p 0.9

多模型路由

# 基于任务类型路由到不同模型
openclaw config set agent.routing.code "gpt-4"
openclaw config set agent.routing.creative "claude-2"
openclaw config set agent.routing.factual "gpt-3.5"

成本控制

# 设置每日预算
openclaw config set agent.budget.daily 10.00

# 启用成本警告
openclaw config set agent.budget.warn_at 80%

故障排除

常见问题

  • 模型不可用: 检查 API 密钥是否有效，网络连接是否正常
  • 配额限制: 某些模型可能有使用限制，检查提供者的配额政策
  • 响应超时: 调整超时设置或尝试更轻量级的模型
  • 本地模型无法连接: 确保本地模型服务正在运行且端口正确

调试命令

# 查看详细日志
openclaw models list --debug

# 测试 API 连接
openclaw models providers --test

# 查看配置状态
openclaw config get agent.model
