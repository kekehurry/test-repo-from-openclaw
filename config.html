config 命令

非交互式配置管理工具，用于获取、设置和取消配置项

命令概述

config 命令提供了一种非交互式的方式来管理 OpenClaw 的配置。它可以用于脚本自动化、CI/CD 流程或批量配置管理。

注意

如果不带任何参数运行 openclaw config，它将启动交互式配置向导（等同于 openclaw setup）。

子命令

config 命令支持以下子命令：

  • config get <key> - 获取指定配置项的值
  • config set <key> <value> - 设置配置项的值
  • config unset <key> - 删除配置项
  • config list - 列出所有配置项
  • config reset - 重置配置到默认状态

常用配置项

以下是一些常用的配置项及其说明：

配置项 说明 示例值
gateway.port Gateway WebSocket 端口 18789
agent.model 默认 AI 模型 openrouter/auto
channels.telegram.token Telegram Bot Token 123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ
workspace.path 工作区路径 ~/.openclaw/workspace

使用示例

获取配置项

# 获取 Gateway 端口
openclaw config get gateway.port

# 获取所有配置
openclaw config list

# 获取特定渠道的配置
openclaw config get channels.telegram

设置配置项

# 设置 Gateway 端口
openclaw config set gateway.port 19000

# 设置默认模型
openclaw config set agent.model "gpt-4"

# 设置工作区路径
openclaw config set workspace.path "/opt/openclaw"

删除配置项

# 删除 Telegram 配置
openclaw config unset channels.telegram

# 重置所有配置
openclaw config reset

环境变量覆盖

OpenClaw 支持通过环境变量覆盖配置项。环境变量的命名规则是将配置项的点号替换为下划线，并转换为大写：

# 等效于 openclaw config set gateway.port 19000
export OPENCLAW_GATEWAY_PORT=19000

# 等效于 openclaw config set agent.model "claude-3-opus"
export OPENCLAW_AGENT_MODEL="claude-3-opus"

在 Docker 中使用

docker run -e OPENCLAW_GATEWAY_PORT=19000 \
           -e OPENCLAW_AGENT_MODEL="gpt-4" \
           openclaw:latest

配置文件位置

OpenClaw 的配置文件通常位于以下位置：

  • Linux/macOS: ~/.openclaw/config.json
  • Windows: %USERPROFILE%\.openclaw\config.json
  • 自定义位置: 通过 OPENCLAW_CONFIG_PATH 环境变量指定

安全提示

配置文件可能包含敏感信息（如 API tokens）。请确保文件权限正确设置，避免意外泄露。
